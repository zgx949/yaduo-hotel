背景介绍：一个酒店对应多个房型，一个房型对应多个策略

# 上架酒店
接口：taobao.xhotel.add
完整文档：https://open.alitrip.com/docs/api.htm?apiId=22665
必填入参：
- outer_id：外部酒店ID, 这是卖家自己系统中的ID（同步亚朵`chainId`）
- name：酒店名称,国内酒店请传中文名称（同步亚朵name）
- city: 标准城市编码参考表 http://hotel.alitrip.com/area.htm?tbpm=3
- address：酒店地址。长度不能超过255。不填入会导致（同步亚朵的`address`）
- tel：酒店电话。格式：国家代码（同步亚朵的`tel`）
- shid：匹配到的酒店id（手动输入或者维护表格）

# 上架房型
接口：taobao.xhotel.roomtype.add
完整文档：https://open.alitrip.com/docs/api.htm?apiId=22662
必填入参：
- out_hid: 对应的酒店outer_id（酒店outer_id=亚朵chainId）
- name：房型名称。不能超过200个字符（同步亚朵`roomTypeName`）
- bed_type：（已废弃）床型请使用bed_info字段(大床/双床)
- outer_id：卖家房型ID，不能重复建议格式是:酒店code_房型code(系统自动生成，亚朵 `chainId_roomTypeId`)
- srid：匹配到的酒店房型id（手动输入或者维护表格）

# 上架策略
接口：taobao.xhotel.rateplan.update
完整文档：https://open.alitrip.com/docs/api.htm?apiId=22676
必填入参
- rateplan_code: 商家价格政策编码(自动生成)
- name: 在淘宝搜索页面展示的房价名称；（添加RP时为必须）。注意该名称不要包含早餐相关信息，如果想维护早餐信息，请设置breakfast_count字段即可。
- breakfast_count：-1,状态早餐，和入住人数有关系，几人价就是几份早餐；0：不含早，1：含单早，2：含双早N，：含N早（1-99可选）；（添加RP时为必须）
- out_hid：外部酒店id=酒店outer_id
- out_rid：外部房型id=房型outer_id
- cancel_policy_cal：取消政策举例[{"date":"yyyy-MM-dd","startDate":"yyyy-MM-dd","endDate":"yyyy-MM-dd","cancel_policy":{"cancelPolicyType":1} },{"date":"yyyy-MM-dd","startDate":"yyyy-MM-dd","endDate":"yyyy-MM-dd","cancel_policy":{"cancelPolicyType":4,"policyInfo":{"48":10,"24":20}}}]
```text
在新增rateplan的同时新增取消政策日历。 json格式。 date：日历的某一天，格式为"yyyy-MM-dd" cancel_policy：日历某一天的价格政策。格式和限制同cancel_policy。 如果date为空，那么会读取startDate和endDate（格式都为"yyyy-MM-dd"），即取消政策属于某一个时间段。 注意：支付类型为预付，那么可以使用所有的退订类型，但是必须是非担保；支付类型为面付或者信任住并且是无担保，那么只能使用1类型的退订；支付类型为面付或者信任住并且为担保，那么只能使用2,5类型的退订；支付类型为在线预约，那么只能使用1,2,5类型的退改。如果支付类型是面付或者信任住并且为担保，那么如果传了4或者6的退订，那么会强制转成类型5。请注意，该字段仅能维护从当前时间开始，10年以内的数据，如果超过10年，会报错。
```
- guarantee_type： 担保类型，只支持： 0 无担保 ，1 峰时首晚担保， 2峰时全额担保， 3全天首晚担保， 4全天全额担保
